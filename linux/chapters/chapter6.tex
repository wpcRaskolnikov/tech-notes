\subsection{脚本格式}
在 Shell 脚本 的第一行通常会写上\verb|#!/bin/bash|，
这行语句称为 shebang，告诉操作系统在执行脚本时，应该调用哪一个解释器。

\subsection{变量}
在 Shell 中，变量可以像 Python 一样直接赋值，不过这种方式定义的变量都是 全局变量；
如果需要定义局部变量，则必须使用 local 关键字。
除了直接赋值，还可以通过 read 从标准输入读取用户输入并赋值，一般配合 echo -n 给出提示，
也可以通过\verb|$0、$1|等读取命令行参数，\verb|$#|表示参数的个数。

在需要向命令传递输入时，Shell 还提供了 两种重定向方式：here string和here document。
Here String 使用 \verb|<<<|，可以把一行字符串作为标准输入传递给命令；
Here Document则适合多行输入，它把分隔符之间的所有文本传递给命令的标准输入。
\begin{verbatim}
# 其中 EOF 是分隔符，可以自定义，只要首尾一致即可。
cat <<EOF
这是第一行
这是第二行
EOF
\end{verbatim}

\subsubsection{搜索}
\begin{itemize}
	\item 行内搜索：fa就是移动到行内下一个a所在之处，使用;能继续搜索。
	\item 全局搜索：/进行全局搜索，使用n继续搜索。
	\item 行内替换：\verb|:s/Line/line/g|，s是替换命令，g代表找到的结果全部替换。
	\item 全局替换：命令为\verb|%s/Line/line/g|，\verb|%|代表搜索整个文件，
	也可以写成\verb|1,$|，代表从第1行到最后一行。
\end{itemize}
